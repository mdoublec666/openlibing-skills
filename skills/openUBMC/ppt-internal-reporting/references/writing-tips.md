# 内容编写经验参考

> 从 SKILL.md 按需引用。实战中总结的编写经验和避坑指南。

## 标题避坑

标题要让不了解项目的读者一眼看懂，把核心判断直接写进标题。

原则：
- 标题超过 25 个字就检查能否精简
- 用"——"破折号做转折比长句更有力
- 写判断句，不要写栏目名（好："XX市：全国首套依托AI的巡逻平台" vs 差："路线图"）

## 禁用词汇

读者看不懂的词汇必须替换：抽象中枢名 → 功能描述、认知术语 → 大白话、造词/自造概念 → 具体描述、内部黑话 → 读者视角、过度渲染 → 事实收住、未翻译英文 → 中文翻译。

## 多页叙事去重

制作完初稿后必须做**跨页去重检查**：
- 如果两页讲的是同一件事的不同写法（如"技术难点"和"竞争壁垒"），合并成一页
- 如果某页的内容已被其他页的框图/表格覆盖，直接删除
- 去重后页码要全部同步更新

## 用户分层一致性

如果定义了关键分类（如用户分层、产品阶段、技术分层），必须确保：
- 所有页面使用**完全相同的名称和描述**
- 架构图、表格、流程图、商业模型中的术语不能自相矛盾
- 修改任一处定义后，必须全文检查其他页面是否同步

## 商业模型页必讲

商业模型页不能只列收入来源，还必须讲清：
1. **成本结构**：核心成本由谁承担（用户 vs 平台 vs 供应商），加价比是多少
2. **资产属性**：产出是否沉淀为固定资产（零边际成本 vs 每次重新消耗）
3. **飞轮效应**：规模增长如何降低单位成本、促进供给增长
4. **收益预估**：至少给出 3 年保守场景的关键指标（用户量/付费率/ARPU/流水/平台收入）

## 壁垒页要有竞争假想

底部总结卡片不要只说"抄不走"，要**假想具体竞争对手**来分析：
- "如果只抄界面" → 为什么用户不会跟着走
- "如果只抄技术方案" → 为什么拿到的是过时版本
- "假设行业巨头来做" → 有什么是砸钱也解不了的（如需要长期积累的数据、关系网络、运营经验）

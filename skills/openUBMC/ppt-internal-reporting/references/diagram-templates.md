# 框图模板参考

> 从 SKILL.md 按需引用。共 7 种可复用框图类型，所有颜色从华为官方色彩表取值（见 `references/color-palette.md`）。

## 通用规则

- 用 CSS Flexbox 或 Grid 布局，不用绝对定位
- 方块用 `border: 1px solid #B5B5B5` 或 `border: 1px solid #C8102E`
- 方块内文字居中，用 `<p>` 标签，不裸放文字在 `<div>` 中
- 箭头用 `→`（文字箭头）或 CSS 三角形，不用图片

---

## 框图 1：水平流程图

适用于：业务流程、操作步骤、审批链路

```
[图标]    →    [图标]    →    [图标]    →    [图标]
 标签           标签           标签           标签
```

HTML 要点：

- `3-8` 个步骤横排，每步由图标 + 标签组成，步骤间用 `→` 箭头连接
- 外层 `display: flex; align-items: center; justify-content: center; gap`
- 每个步骤是 `flex-direction: column` 的小容器（图标在上、标签在下）
- 箭头是独立的文字元素 `→`，颜色用深灰 `#595757`
- 图标可用 Flaticon 下载图标、Unicode 符号、简单 SVG 或 CSS 形状
- 图标区域统一大小（如 `48px × 48px`），加圆角或圆形底色

CSS 配色：

- 图标背景：极浅灰 `#DDDDDD` 或白底 + 浅灰边框
- 图标如需强调当前步骤：华为红底 + 白色图标
- 标签文字：近黑 `#231815`

## 框图 2：架构堆叠图

适用于：技术架构、平台分层、能力矩阵

```
┌─────────────────────────────────────────┐
│  ┌──────────┐  ┌──────────────────────┐ │  ← 层内多块并排
│  │  模块 A   │  │      模块 B          │ │
│  └──────────┘  └──────────────────────┘ │
├─────────────────────────────────────────┤
│            整层单一模块                   │  ← 层内单块通栏
├─────────────────────────────────────────┤
│  ┌────┐  ┌────┐  ┌────┐  ┌────┐       │  ← 层内多块均分
│  │ C  │  │ D  │  │ E  │  │ F  │       │
│  └────┘  └────┘  └────┘  └────┘       │
└─────────────────────────────────────────┘
```

HTML 要点：

- `3-5` 层矩形横条从上到下堆叠，每层可包含多个并排方块
- 外层容器设固定宽度，内部每层是一个 `div`
- 层内多块用 `display: flex; gap` 排列
- 整层单块不需要内部 flex
- 层与层之间 `gap: 4px` 或 `margin-top: 4px`

CSS 配色：

- 每层底色从上到下逐渐变浅：最上层用极浅灰 `#DDDDDD`，下层用更浅或白底
- 或反过来：底层（基础设施）用深底色，上层（应用）用浅底色
- 方块边框：浅灰 `#B5B5B5`
- 方块内文字居中，近黑 `#231815`
- 堆叠图整体可加标题行，标题用近黑粗体居中

## 框图 3：方案标题横幅

适用于：解决方案名称、产品名称、系统名称的突出展示

```
╭──────────────────────────────╮
│     方案名称 / 产品名称       │
╰──────────────────────────────╯
```

HTML 要点：

- 单个 `<div>` 或 `<p>`，`text-align: center`
- `border: 2px solid #C8102E; border-radius: 20px; padding: 8px 32px`
- 文字华为红 `#C8102E`，加粗
- 宽度 `fit-content`，居中于父容器
- 通常放在流程图上方或页面中部作为锚点

---

## 框图 4：优势特性卡片行

适用于：核心优势展示、关键特性并列、差异化卖点

```
┌────────────┐  ┌────────────┐  ┌────────────┐
│    [图标]    │  │    [图标]    │  │    [图标]    │
│  红色标题    │  │  红色标题    │  │  红色标题    │
│  灰色描述    │  │  灰色描述    │  │  灰色描述    │
└────────────┘  └────────────┘  └────────────┘
```

HTML 要点：

- `3` 张卡片横排，每张包含图标 + 红色标题 + 灰色描述
- 外层 `display: flex; justify-content: center; gap`
- 每张卡片 `flex-direction: column; align-items: center; text-align: center`
- 图标区域统一大小
- 标题：华为红 `#C8102E`，加粗
- 描述：深灰 `#595757`，字号小于标题

## 框图 5：水平节点条

适用于：合作阶段、能力维度、关系链展示

```
                标签B            标签C
    标签A    ●───────────●───────────●    标签D
  ●━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━●
```

HTML 要点：

- 一条水平条带上 `3-5` 个红色圆点标注关键节点，每个节点上方或下方有标签
- 底层是一个水平条 `div`：华为红背景，高度 `4-6px`，`border-radius: 3px`
- 节点是绝对定位或 flex 均分的圆点：`width: 12px; height: 12px; border-radius: 50%; background: #C8102E`
- 标签用相对定位放在节点上方或下方，近黑色
- 条带两端可以有端点标签

## 框图 6：金字塔层级图

适用于：组织架构、能力层级、资源层级（从基础到顶层）

```
         ┌──────────┐
         │  顶层聚焦  │
      ┌──┴──────────┴──┐
      │    中层能力      │
   ┌──┴────────────────┴──┐
   │      底层基础设施       │
   └──────────────────────┘
```

HTML 要点：

- `3-4` 层从底部宽到顶部窄的水平条，形成梯形/金字塔
- 每层是一个 `div`，宽度递减（如底层 `100%`、中层 `75%`、顶层 `50%`）
- 所有层居中对齐：`margin: 0 auto`
- 层间 `gap: 4px`
- 每层可内含多个并排方块（同架构堆叠图）
- 如需梯形效果，可用 `clip-path: polygon(...)` 裁切，但简单版用不同宽度矩形即可

CSS 配色：

- 底层颜色深、顶层颜色浅，或使用红色渐变层级
- 可以底层用极浅灰，中层用浅灰，顶层用华为红底白字

## 框图 7：底部荣誉认证条

适用于：奖项、认证、资质、合作方的底部声明

```
🏆  国际物流与运输学会（CILT）：降低全社会物流成本"金钥匙"典型案例奖
```

HTML 要点：

- 单行 `display: flex; align-items: center; justify-content: center; gap: 8px`
- 图标用 Unicode（🏆🏅🎖️📜）或简单 SVG
- 文字华为红 `#C8102E`，可加粗
- 放在页面最底部，品牌 logo 行之上
- 与底部高亮统计条功能类似但内容偏"荣誉"而非"数据"

## 框图组合规则

实际页面中通常**纵向组合**多种框图。常见的组合顺序：

1. 方案标题横幅 → 水平流程图 → 优势特性卡片行 → 架构堆叠图 → 底部荣誉条
2. 水平节点条 → 大数字横排指标条 → 图标成果卡片行
3. 金字塔层级图 + 左右两栏文字说明

组合时注意：

- 框图与框图之间留足垂直间距（至少 `16-24px`）
- 所有框图居中于页面内容区
- 一页内不超过 `2-3` 种框图类型，避免视觉混乱
- 框图中文字密度要低于正文区——框图靠结构说话，不靠堆文字

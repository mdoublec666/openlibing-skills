# PPT Internal Reporting（华为式管理汇报风格）

## 何时使用

优先在这些场景使用：

- 用户说"华为风"，但内容是战略、产品、技术、路线图、管理汇报
- 用户需要高知识密度、结论先行、偏决策文书感的页面
- 用户要求页面更像"给听众看"，而不是"给主讲人自己看"
- 用户要求去掉过程态、脚手架、制作说明、讲者提示
- 用户给了华为管理汇报截图，要求照此风格做

## 核心判断

这种风格不是发布会炫技风，而是**华为式管理汇报风格**：

- 重点是结论、结构、决策效率
- 不是重点做舞台感、产品大图和炫目的视觉效果
- 强调靠信息组织和版式秩序，不靠装饰堆砌
- 封面页、章节分隔页、收束页按一般 PPT 处理即可，不需要特殊规则
- 多页叙事节奏不做硬性约束，根据内容自然编排

## 标题规则

- 标题优先写成**判断句**，把阶段、规模、数量或交付结果直接写进去
- 标题用**华为红**，字号要够大，在页面中形成第一视觉焦点
- 允许用"冒号前缀"指定范围：`XX市：全国首套依托XX的平台，硕果累累`
- 不写成中性栏目名

好例子：

- `SZ市：全国首套依托视觉AI代替人力巡逻平台，硕果累累`
- `深圳职业技术大学：课证共生共长，产教协同育人`
- `长沙市数字技能人才公共实训基地：服务湖南"三高四新"战略，打造区域数字人才新高地`
- `路线顺序必须是先作品、后共建、再平台`

差例子：

- `路线图`
- `产品定位`
- `技术深度`

### 结论摘要段

标题下方可以紧跟一段 `1-3` 行的结论摘要段，直接用数据支撑标题判断：

- 正文用黑/深灰
- 关键数字、首创/里程碑等关键词用**红色加粗**
- 让读者不展开下方内容也能抓住核心结论

示例：`依托AI视觉大模型，启动部署5个月后，已支撑超**30000路**智能化视频巡逻，上报打架斗殴**1500+起**。`

## 强调原则（加粗标红）

华为汇报风的核心视觉手法是**标题够大、子标题够大、重点加粗标红**：

- 标题和区块子标题的字号必须显著大于正文，形成清晰的视觉层级
- 凡是需要读者注意的关键词、关键数据、关键结论，一律用**华为红 + 加粗**
- 不要把所有文字都做同一个字号——没有层级 = 没有重点
- 宁可多标红几个关键词，也不要让读者自己去猜哪里是重点

### 字号层级（pt）

| 层级 | 字号范围 | 说明 |
|------|---------|------|
| 页面主标题 | `28-32pt` | 红色加粗判断句，一页只有一个 |
| 区块子标题 | `18-22pt` | 红色粗体，如 `▷ 业务挑战` |
| 正文 | `12-16pt` | 近黑色，根据信息密度在此区间调整 |
| 大数字指标 | `36-48pt` | 红色，横排指标条或底部高亮统计条中的数字 |
| 框图内文字 | `8-12pt` | 架构图、流程图方块内的标签，最小不低于 `8pt` |
| 注释 / 脚注 | `8-10pt` | 灰色，页面底部备注信息 |

字号选择原则：

- 同一页内至少保持 **3 个明显层级**（标题 / 子标题 / 正文）
- 标题与正文之间字号差至少 `12pt`
- 框图内文字虽然小，但必须保持可读性，低于 `8pt` 一律禁止

## 页面规则

- 一页只表达一个核心判断
- 结构默认是：标题判断 -> 结论摘要段（可选） -> 主体区 -> 收束区
- 主体区优先从下方**页面原型**中选一种，而不是平均铺满 bullet
- 证据区优先用表格、矩阵、对比块、关键事实卡
- 有数据支撑时遵循：`字不如表，表不如图`
- 能用柱状图、饼图、对比图、关系图说明，就不要退回长段落
- 页眉只保留必要信息，例如品牌和页码
- 不要在页面里保留给制作人或讲者看的元信息

## 常用页面原型

根据页面要表达的内容，优先从以下原型中选择。不要每页都套同一种。

### 原型 A：左结构 / 中心对象 / 右证据表

适用于：能力架构页、机制页、核心产品页

- 左侧负责解释参与方、机制、层级、阶段或关系
- 中间核心对象只保留 `2-4` 条稳定价值或核心能力，不堆过程话术
- 右侧表格负责回答"谁在参与、交付了什么、证明了什么结果"

### 原型 B：诉求-方案-价值 三栏并列

适用于：案例成果页、项目汇报页、解决方案总览页

三栏布局，用细竖线分隔，每栏顶部用**红色区块标题**（斜体可选）。

有两种常见比例：

- **等宽三栏**：三栏均分，适合三部分内容量相当
- **中栏加宽**（推荐）：左窄 / 中宽 / 右中，适合中栏放架构图或流程图

栏头名称可选变体：

- `业务挑战 / 解决方案 / 效果`
- `客户诉求 / 解决方案 / 价值呈现`
- `现状痛点 / 能力建设 / 成果输出`

各栏内容填充：

- 左栏（诉求/挑战）：先用 `1-2` 句简介交代背景主体（关键实体名加粗），再用 ▷ 标记逐条列痛点，每条格式为 `▷ **粗体关键词**：描述文字`
- 中栏（方案）：放架构图、金字塔层级图、流程图或核心能力矩阵
- 右栏（效果/价值）：用 ▷ 标记逐条列价值，每条格式同左栏

页面底部可选加**底部高亮统计条**（见区块组件）。

### 原型 C：现状-差距-目标

适用于：现状分析页、差距诊断页、目标对齐页

- 左栏写当前现状或基线
- 中栏用红色高亮差距、风险或瓶颈
- 右栏写目标状态或目标指标

### 原型 D：对比分析页

适用于：方案对比、竞品对比、前后对比

- 用表格或双栏对比
- 对比维度写成行标题
- 优势项用红色或绿色标注
- 底部写出结论判断，不让读者自己去总结

### 原型 E：路线图 / 里程碑页

适用于：产品路线图、项目计划、阶段回顾

- 水平时间轴，从左到右
- 已完成阶段用实色（灰或蓝），当前阶段用红色高亮
- 每个阶段标注关键交付物和数据指标

### 原型 F：数据大屏 / 关键指标页

适用于：运营结果、KPI 汇报、周/月报核心页

- 顶部 `3-5` 个大数字卡片，每个有指标名、数值、变化趋势
- 下方放趋势图或对比柱状图
- 红色标注关键增长或风险指标

### 原型 G：成果展示页（垂直分层堆叠）

适用于：合作成果展示、项目总结、标杆案例页

页面纵向分成 `3-4` 个水平层带，从上到下依次堆叠：

1. **流程/关系条**：水平条带 + 红色圆点标注关键节点，展示合作内容、流程阶段或关系链
2. **大数字横排指标条**：`3-5` 个红色大数字 + 灰色副标题，用细竖线分隔（见区块组件）
3. **图标成果卡片行**：`2-4` 张横排卡片，每张有图标/图片 + 成果描述（见区块组件）
4. **人物证言卡行**（可选）：`2-3` 张横排人物卡（见区块组件）

每层之间用留白或淡灰分隔线隔开，保持垂直节奏感。

## 区块组件规则

### 区块标题

- 用 ▷ 或 > 作为区块标记符，红色
- 区块标题本身用红色粗体，字号要显著大于正文
- 示例：`▷ 业务挑战`、`▷ 解决方案`、`▷ 效果`

### ▷ 关键词条目列表

- 格式：`▷ **粗体关键词**：后续描述文字`
- 关键词限 `2-4` 字，用红色或黑色粗体均可
- 描述文字中的关键数据仍用**加粗**
- 一栏通常 `4-6` 条
- 示例：`▷ **缺设备**：缺乏数字经济类工种培训设备`、`▷ **促产业**：旨在抢占产业、技术、人才、平台四个高点`

### 编号痛点 / 要点列表

- 用圆圈数字 ①②③ 编号
- 每条的标题用**红色粗体**，限 `4-8` 字
- 标题下方跟 `1-2` 行黑色正文，说明具体内容
- 一栏通常不超过 `3-4` 条

### 效果 / 价值卡片

- 每张卡片由**红色粗体关键词**（`2-4` 字）+ 正文描述组成
- 正文中的关键数据用**加粗**
- 可以加小图标辅助，但不必须
- 一组通常 `3` 张，纵向排列
- 示例：`防风险` — "从苗头、隐患阶段及时控制，大模型识别打架斗殴行为告警准确率达**90%**"

### 大数字横排指标条

- 一行横排 `3-5` 个指标，用细竖线分隔
- 每个指标：红色大号数字（如 `5000+`、`10万+`）+ 下方灰色小字说明
- 数字用远大于正文的字号，形成视觉锤
- 适合放在页面中部或原型 G 的第二层

### 图标成果卡片

- 横排 `2-4` 张，每张有左侧图标/图片 + 右侧文字描述
- 文字中的关键成果用**加粗**
- 示例：书籍图标 + "主编1+X网络系列教材获批**国家十三五规划教材**"
- 适合展示奖项、资质、里程碑事件

### 人物证言卡

- 横排 `2-3` 张，每张包含：头像照片 + 姓名（红色粗体）+ 资质认证 + 关键成就
- 关键成就中的亮点（如竞赛名次、职位）用**加粗**
- 用于展示标杆人物、合作伙伴代表或客户证言
- 卡片间距均匀，底部对齐

### 架构分层图

- 用于解决方案或技术架构展示
- 两种常见变体：
  - **横向分层**：SaaS / PaaS / IaaS 或 应用层 / 平台层 / 基础设施层，从上到下
  - **金字塔层级**：三平台 → 二平台 → 一基地，从底部到顶部，越往上越聚焦
- 每层用不同深浅的背景色区分
- 层内用小方块列出关键能力或模块
- 层间用箭头或连线表示关系

### 底部高亮统计条

- 一行居中文字，嵌入一个超大红色数字
- 示例：`每年培养数字技能人才 **9000+** 人次`
- 数字字号显著大于周围文字（约 `2-3` 倍）
- 放在页面最底部（品牌 logo 行之上），作为全页收束
- 用于强化单一核心指标

## 框图绘制规则

华为管理汇报页中的框图是结构化表达的核心手段。共 **7 种可复用框图类型**（水平流程图、架构堆叠图、方案标题横幅、优势特性卡片行、水平节点条、金字塔层级图、底部荣誉认证条），详细的 HTML/CSS 模板和组合规则见 `references/diagram-templates.md`。

通用原则：
- 用 CSS Flexbox 或 Grid 布局，不用绝对定位
- 方块用 `border: 1px solid #B5B5B5` 或 `border: 1px solid #C8102E`
- 方块内文字居中，用 `<p>` 标签，不裸放文字在 `<div>` 中
- 箭头用 `→`（文字箭头）或 CSS 三角形，不用图片
- 所有颜色从华为官方色彩表取值（见 `references/color-palette.md`）
- 一页内不超过 `2-3` 种框图类型，框图间留足垂直间距（至少 `16-24px`）
- 框图中文字密度要低于正文区——框图靠结构说话，不靠堆文字

### 图标选择原则

在 HTML-first 工作流中，框图内的图标按以下优先级选择：

1. **Flaticon 下载图标**（首选）：通过 `flaticon-downloader` skill 搜索并下载高质量 PNG/SVG 图标，放在项目的 `icons/` 目录下。适合需要统一风格、专业感的图标场景（如能力卡片、流程图节点、优势特性图标等）。调用方式：
   - 用 `web_search` 搜索 `site:flaticon.com/free-icon <关键词>`
   - 从搜索结果中提取图标 URL
   - 用 `flaticon_downloader.py` 批量下载到 `icons/` 目录
   - 在 HTML 中用 `<img src="icons/xxx.png">` 引用
   - 同一组页面内的图标保持**相同风格**（outline/fill）和**相同尺寸**
2. **Unicode 符号 / Emoji**：快速原型或辅助场景（📋📝✅📊🎯🔍📦🏆💡🔒）
3. **CSS 纯形状**：用 `border`、`border-radius`、`clip-path` 画简单几何图形
4. **内联 SVG**：需要精确图标但无法通过 Flaticon 获取时，手写简单 SVG path

**经验法则**：正式交付的 PPT，框图/卡片中的图标优先走 Flaticon；只在 Flaticon 找不到合适图标时，才退回 Unicode/Emoji 或 CSS 形状。

### 图片资源与占位规范

页面中需要照片、插画、产品截图等**非图标类图片资源**时，按以下流程处理：

#### 第一步：判断是否需要图片

以下场景通常需要图片：
- 产品展示页（产品截图、实物照片）
- 案例成果页（现场照片、交付物照片）
- 封面页（城市剪影、科技背景等装饰图）
- 人物证言卡（头像照片）
- 数据大屏页（图表截图）

#### 第二步：使用灰色占位块预留位置

如果当前没有用户提供的图片资源，**不要跳过图片区域**，而是用灰色占位块预留位置：

```html
<!-- 图片占位块 -->
<div style="width:200pt;height:150pt;background:#DDDDDD;border-radius:4pt;
            display:flex;align-items:center;justify-content:center;flex-direction:column;gap:4pt;">
  <span style="font-size:10pt;color:#898989;">📷</span>
  <span style="font-size:8pt;color:#898989;text-align:center;line-height:1.3;
              padding:0 8pt;">产品截图<br>（等待用户提供）</span>
</div>
```

占位块规范：
- 背景色：极浅灰 `#DDDDDD`
- 圆角：`4pt`
- 内部居中放置 📷 emoji + 灰色描述文字
- 描述文字要写清楚**期望的图片内容**（如"XX平台产品截图"、"深圳基地现场照片"）
- 占位块的尺寸要和最终图片尺寸一致，避免替换后布局塌陷

#### 第三步：提示用户

完成初稿后，在预览页面中明确告知用户：
> "以下位置预留了图片占位，请提供对应的图片资源：
> - 第 3 页：XX平台产品截图（建议尺寸 400×300）
> - 第 5 页：人物头像照片（建议尺寸 200×200）
> - 第 7 页：城市剪影背景图（建议尺寸 960×540）
> 
> 如果不提供图片，我可以删除占位区域并重新调整布局。"

#### 第四步：用户响应后处理

- **用户提供了图片**：替换占位块为 `<img>` 标签，保持尺寸不变，调整 `object-fit` 确保图片不变形
- **用户表示不提供**：删除占位块，重新编排该区域的布局（缩小其他元素间距、拉宽文字区域等），确保视觉不空洞
- **用户提供了但尺寸不匹配**：用 `object-fit: cover` + `overflow: hidden` 裁切，或按比例缩放

## 表格样式规则

- **表头行**：有背景填充（华为红 `#C8102E` 底 + 白色文字，或极浅灰 `#DDDDDD` 底 + 近黑文字），居中，加粗，字号比表体大 `1-2pt`
- **表体行**：白底 + 近黑文字，居中或左对齐视内容而定
- **交替行底色**（可选）：偶数行用极浅灰 `#DDDDDD`，增强可读性
- **边框**：细实线，浅灰 `#B5B5B5`，不用粗边框
- **单元格内间距**：上下 `4-6px`、左右 `8-12px`，不要挤满
- **关键数据**：表体内的重点数字仍用华为红加粗
- 表格整体居中于所在栏或页面，不要左对齐贴边

## 数据呈现规则

- 正文中的关键数字用**华为红加粗**：`警情下降**16.5%**`
- 大标题和结论摘要段中的数据也用红色加粗
- 百分比、金额、时间周期、数量等优先用数字而非文字
- 首创 / 唯一 / 第一 等关键定性词也可红色加粗
- 数据必须有比较基准（同比、环比、部署前后），不要裸数字

## 色彩规范

所有颜色必须使用华为官方色值，不要自行猜测。完整色表见 `references/color-palette.md`。

核心速查：
- **华为红**（主强调色）：`#C8102E` — 标题、关键词、强调数据
- **近黑**（正文主色）：`#231815`
- **深灰**（次要文字）：`#595757`
- **浅灰**（边框/分隔）：`#B5B5B5`
- **极浅灰**（背景/底块）：`#DDDDDD`
- 页面上 90% 以上面积只出现**华为红 + 近黑 + 深灰 + 白色**
- 暖色系（橙、金）仅在图表多系列区分时使用，正文和标题不用

## 视觉规则

- 白底
- 华为红 `#C8102E` 作为单一强调色（标题、区块标记、高亮数据、关键词）
- 正文用近黑 `#231815`，次要信息用深灰 `#595757`
- 分隔线、细边框用浅灰 `#B5B5B5`，背景底块用极浅灰 `#DDDDDD`
- 栏间用细竖线分隔，不用粗边框
- 信息密度高，但不能做成说明书
- 不用噪点、纹理、科技花纹、装饰性渐变
- 不为了"高级感"额外加科技装饰
- 允许在页面角落放**极弱装饰性背景图**（如城市剪影、建筑轮廓），半透明，绝不抢主内容
- 表格和卡片之间留足间距，避免挤压感
- 品牌 logo 放右下角，不占用内容区

## 听众页禁用内容

这些内容默认不要直接出现在页面里：

- `本页结论`
- `最终结论`
- `Step 1 / Step 2`
- `Layer 1 / Layer 2`
- `Phase 1 / Phase 2`
- `这份新稿的变化`
- `建议使用场景`
- `下一步可继续做`
- 只有概念没有证据的空泛大词
- 把高知识密度误做成满屏 bullet 堆砌

如果确实需要这些信息，把它们放到讲稿、备注或单独文档，不放在 slide 正文。

## 页面拆分原则

- 如果一页同时在定义概念、讲路线、讲商业、讲风险，说明页内负担过重
- 宁可拆页，也不要把一页做成文档摘要
- 封面页不做目录页
- 结尾页做收束，不做制作说明

## 页面完成标准

一页做完后用三个词自检：

- **结果感**：标题和摘要段是否直接告诉读者"结果是什么"
- **结构感**：主体区是否有清晰的分栏/分区/层次，而不是平铺 bullet
- **证据感**：是否有具体数据、表格、对比或案例支撑结论

## 内容编写经验

实战中总结的标题避坑、禁用词汇、去重检查、商业模型页、壁垒页等经验，见 `references/writing-tips.md`。编写时关键记住：

- 标题写判断句，不要写栏目名
- 禁止读者看不懂的术语和自造词
- 完稿后必须做跨页去重检查
- 商业模型页要讲清成本结构、资产属性、飞轮效应、收益预估
- 壁垒页要假想具体竞争对手

能力验证

自迭代能力的四个具体体现

🔁

失败自愈

compile_omni 失败 → 读编译错误 → 定位问题 → 修改代码 → 重编译；q3 运行期报 MakeDecimal not found → 定位 vector path 缺失 → 三轮递进修复

📈

证据驱动决策

每次性能优化前必须先采集火焰图；Opt3 关键突破（JNI 1000→1次）来自火焰图中 GetOutput 热点，而非猜测。硬性规则：没有画像不允许改代码。

📄

跨轮次记忆

每轮迭代的中间数据、当前 ratio、下一步方向全部通过文档持久化（modification_trace、perf report）。不依赖上下文窗口，状态显式管理。

🔒

卡点有效性

Skill 中的强制文档卡点被实际遵守：context_sortagg.md 与 self_validation_sortagg.md 先于任何实现代码存在且内容完整。卡点不是建议，是约束。

功能覆盖亮点

  • PARTIAL / FINAL / SINGLE / COMPLETE 全覆盖
  • collect_set / collect_list / COUNT / SUM / AVG / MIN / MAX

已知遗留 Gap（诚实列出）

  • collect_set(MAP 元素) — 实现复杂度高
  • Spill 路径未专项验证
  • 复杂类型 ARRAY/ROW 作为 group-by key

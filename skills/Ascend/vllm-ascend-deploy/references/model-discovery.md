## 核心原则

**先问用户，再搜索；不硬编码、不假设**

1. ✅ **优先询问用户**：用户提供的配置直接使用
2. ✅ **按优先级搜索**：用户未提供时从官方渠道查找
3. ✅ **网络受限时用浏览器**：web_fetch 失败 → browser 工具
4. ❌ **不硬编码模型映射表**：会快速过时

## 发现流程

```
用户请求: "部署 <模型名>"
    ↓
Step 1: 询问用户
    - 有指定镜像路径？
    - 有指定权重路径？
    - 有指定启动参数？
    ↓
用户明确提供具体值 → 直接使用，跳过搜索
    ↓
用户模糊授权（"你自己确定"）→ 视为未提供，继续 Step 2
    ↓
Step 2: 按优先级搜索（用户未明确提供时）
    - GitCode → 官方文档 → Modelers → 华为支持
    ↓
Step 3: 提取配置
    - 镜像（注意 A2/A3）
    - TP 推荐值
    - 启动参数
    ↓
Step 4: 生成确认清单
    ↓
Step 5: 用户确认后部署
```

## 配置提取方法

### 从 GitCode 教程提取

1. 搜索模型名（如 "Qwen3.5-27B"）
2. 打开匹配的项目 README
3. 提取：
   - Docker 镜像名称
   - `--tensor-parallel-size` 值
   - vLLM 启动参数
   - 环境变量

**⚠️ 镜像仓库深度搜索：** 文档示例通常只展示一种架构（如 A3），但镜像仓库往往同时提供 A2/A3 版本。当找不到匹配架构的镜像时，**进入镜像仓库目录查看完整文件列表**，根据 NPU 架构选择匹配版本。

### 关键提取项

| 提取项 | 位置 | 示例 |
|--------|------|------|
| 镜像 | docker load 或 docker run | `vllm-ascend:qwen3_5-v0-a2` |
| TP | --tensor-parallel-size | 4 |
| max-model-len | --max-model-len | 32768 |
| shm-size | --shm-size | 100g |

## 硬件差异处理

### NPU 型号检测

```bash
npu-smi info
# Name 列显示型号
```

**硬件架构对应关系：**

| npu-smi 显示 | 架构 | 物理结构 | 说明 |
|--------------|------|----------|------|
| 8 × 910B3 | A2 | 8卡8die | 每卡1个910B3 |
| 16 × 910B3 | A3 | 8卡16die | 2die（也就是2张910B3）组成1个910C，共8张910C相当于16张910B3 |

**关键：** A3 架构，镜像需用 `-a3` 后缀

### 镜像匹配规则

| npu-smi 显示 | 架构 | 镜像特征 | 示例 |
|--------------|------|----------|------|
| 8 × 910B3 | A2 | `-a2` 或 `-A2` | `vllm-ascend:qwen3_5-v0-a2` |
| 16 × 910B3 | A3 | `-a3` 或 `-A3` | `vllm-ascend:qwen3_5-v0-a3` |

**重要：** 镜像必须匹配 NPU 架构，否则可能无法启动

### TP 推荐值提取

从搜索到的文档中提取，不同模型差异大：

| 模型 | A2 TP | A3 TP | 来源 |
|------|-------|-------|------|
| Qwen3.5-27B | 4 | - | GitCode |
| Qwen3.5-35B-A3B | - | 8 | GitCode |

**不要硬编码**，每次从文档实时提取

## 确认清单模板

**⚠️ 严格遵守：先用官方推荐，再告知用户现有资源情况**

```markdown
找到 {MODEL} 的推荐配置：

## 官方推荐配置

| 参数 | 值 | 来源 |
|------|-----|------|
| 容器镜像 | {OFFICIAL_IMAGE} | {SOURCE} |
| TP 并行数 | {TP} | 官方文档推荐 |
| 模型路径 | {PATH} | 用户指定 |
| 服务端口 | {PORT} | 默认 |

## NPU 资源

- npu-smi 显示: {NPU_NAME} → 架构: {ARCH} (A2/A3)
- 空闲卡: {FREE_COUNT}

## 现有资源检查（仅供参考）

- 已有容器: {EXISTING_CONTAINER} (镜像: {EXISTING_IMAGE})
- 已有镜像: {EXISTING_IMAGES}

## 选择

1. **使用官方推荐镜像**: {OFFICIAL_IMAGE}
2. **复用现有容器/镜像**: {EXISTING_IMAGE}（如可用）

请确认选择（回复 1 或 2）。
```

**禁止：**
- 看到现有容器就直接用，忽略官方推荐
- 不告知用户有选择余地

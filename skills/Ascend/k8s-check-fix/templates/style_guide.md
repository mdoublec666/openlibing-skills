# 输出风格指南

## 通用原则
- 使用清晰的标题层级：`#` 主标题，`##` 二级标题，`###` 三级标题
- 重要状态用 **粗体** 强调
- 使用表情符号增强可读性：
  - ✅ 成功/健康
  - ❌ 失败/错误
  - ⚠️ 警告/注意
  - 🏥 诊断结果
  - 🔧 修复操作
  - 📦 Deployment 相关
  - 🌐 网络相关

## 表格格式
- 表格首行加粗，使用 `|` 分隔
- 数据对齐：左对齐文本，右对齐数字
- 空值使用 `-` 表示

## 代码块
- 命令使用 ```bash 语言标识
- 日志使用 ```text 或直接 ``` 无标识
- 输出内容限制在 200 行以内，过长时截断并注明

## 诊断结论要求
- 必须包含**根本原因**分析
- 必须给出**具体建议**，涉及命令时使用代码块
- 如有多种修复方案，按风险从低到高列出

## 安全提示
- 输出日志前，添加警告：“⚠️ Pod 日志可能包含敏感信息，请谨慎处理。”
- 写操作前，必须展示完整命令并请求确认
- 遇到 RBAC 错误时，提示用户缺少的权限和最小权限配置建议
